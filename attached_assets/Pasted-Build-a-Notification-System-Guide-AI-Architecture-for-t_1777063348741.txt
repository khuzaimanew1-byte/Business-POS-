Build a **Notification System (Guide + AI Architecture)** for the POS app. Follow this as a strict implementation guide. The system must be **actionable-only, anti-spam, auto-cleaning, and performance-optimized**. UI should be premium, minimal, and smooth.

---

# 🔔 🎯 CORE PRINCIPLE

> Only show notifications when the user must take action.
> No success messages. No unnecessary alerts. No repetition.

---

# 🧠 1. SYSTEM ARCHITECTURE (FLOW)

### Event → Filter → Create → Display → Resolve → Remove

```text
Event detected
→ Check if actionable
→ Check if already triggered
→ Create notification
→ Show toast (top)
→ Add to list (page)
→ Wait for action / condition resolve
→ Auto remove
```

---

# ⚙️ 2. EVENT DETECTION LAYER

Trigger only meaningful events:

### ✅ Allowed:

* Product stock ≤ threshold (low stock)
* Product stock = 0 (out of stock)
* System error / conflict

### ❌ Ignore:

* Success (created, saved, updated)
* Minor UI actions
* Background processes

---

# 🧠 3. INTELLIGENT FILTER (ANTI-SPAM)

Before creating notification:

```text
if (actionRequired === false) → ignore
if (alreadyTriggered === true) → ignore
if (cooldownActive === true) → delay
else → allow
```

---

### Per Product Logic:

* Each product has `lowStockTriggered` flag
* Trigger only once
* Reset when stock increases again

---

# 🧾 4. NOTIFICATION OBJECT STRUCTURE

```json
{
  id: unique,
  type: "alert" | "warning",
  message: string,
  actionLabel: string,
  actionTarget: string,
  quickCode: string,
  timestamp: number,
  isRead: false,
  triggered: true
}
```

---
