# ⌨️ SHORTCUT SYSTEM — GLOBAL EXECUTION ENGINE

## 🎯 CORE GOAL

Shortcut keys must:

* Work instantly after editing
* Apply across **all pages**
* Respect context (where needed)
* Never feel broken or inactive

---

# ⚙️ 1. REAL-TIME SHORTCUT APPLICATION (CRITICAL)

## ✅ RULE:

When a shortcut key is edited:

* It must be **applied immediately**
* No reload required
* No “save to apply later” behavior

---

## 🔄 SYSTEM BEHAVIOR:

* Old shortcut → instantly removed
* New shortcut → instantly active
* No duplicate bindings allowed (handle conflict)

---

# 🌐 2. GLOBAL SHORTCUT AVAILABILITY

## ✅ RULE:

Shortcuts should work from **any page**, unless context-specific

---

## 🧠 SHORTCUT TYPES

### 🔹 GLOBAL ACTION SHORTCUTS (ALWAYS WORK)

These must trigger from anywhere:

* Open Analytics (Shift + A)
* Open Add Product (Shift + P)
* Open Settings (if assigned)

👉 These override current page and navigate if needed

---

### 🔹 CONTEXTUAL SHORTCUTS (CONDITIONAL)

These only work if related UI is active:

* Enter → Create product

  * Only works on Add Product Page

* Shift + Enter → Create & Add Another

  * Only on Add Product Page

* Enter (Search Active) → Add product to cart

  * Only when search input is focused

* C + Number + ↑ / ↓

  * Only when cart is open

* Open Cart (Shift + C)
* Toggle Edit Mode (Shift + E)
* Toggle Search (Ctrl + `)
only in main pos page

---

## ❗ CONDITION RULE:

If required component/page is NOT active:

* Shortcut should **NOT trigger action**
* No error shown
* Silent ignore

---

## Example:

## RULE:

Navigation shortcuts:

* Always execute
