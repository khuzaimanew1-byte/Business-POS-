Refine the **Analytics Line Graph System** into a precise, conflict-free, production-level implementation with strict time alignment, correct data logic, and an advanced “future zone” UX.

# 📊 1. TIME RANGE SYSTEM (STRICT EDGE ALIGNMENT)

## 🎯 GOAL:

Graph must ALWAYS:

* Start exactly at selected **start time/date**
* End exactly at selected **end time/date**

👉 No visual gaps on left or right

---

## ⚙️ IMPLEMENTATION RULE

### X-Axis Domain:

```text
[minTime, maxTime]
```

* `minTime` = exact start (daily/hourly/custom)
* `maxTime` = exact end

---

## ❗ FIX CURRENT ISSUE (WEEKLY / MONTHLY GAP)

### Problem:

* Slight padding/gap on edges

### Fix:

* Disable chart library padding:

  * No `offset`
  * No `boundaryGap`
  * No auto margins

👉 Force:

* First point = touches start edge
* Last point = touches end edge

---

# 📈 2. DATA GENERATION LOGIC (EVENT-BASED + SMART BASELINE)

---

## 🧠 CORE RULE:

* Data is **event-driven**
* But graph rendering is **continuous**

---

## 🔹 CASE A: START HAS NO ACTIVITY

### Required:

If no activity at start:

👉 Inject artificial starting point:

```text
(startTime, 0)
```

---

## 🎯 PURPOSE:

* Graph visually starts from baseline
* Tooltip must work at start

---

## 🔹 CASE B: AFTER START → FIRST ACTIVITY

```text
(start → flat at 0 → jump to first value)
```

---

## 🔹 CASE C: BETWEEN ACTIVITIES

If gap exists:

```text
(lastValue → flat → nextValue)
```

👉 Never drop to 0

---

## 🔹 CASE D: MULTIPLE EVENTS

* Allow multiple timestamps
* Do NOT merge aggressively unless needed

---

## 🔹 CASE E: END WITH NO ACTIVITY

* Extend last value till `maxTime`

---

# 📍 3. TOOLTIP LOGIC

---

## 🎯 RULES:

### At START (even if artificial):

* Tooltip must show:

  * Time
  * Value = 0

---

### At REAL POINT:

* Show:

  * Exact timestamp (with minutes)
  * Value (sales/profit)

---

### BETWEEN POINTS:

* Optional:

  * Interpolated tooltip OR no tooltip (cleaner)

---

# 📉 4. LINE RENDERING RULE

---

## ✅ LINE MUST:

* Be continuous
* Never break
* Never drop to zero unless actual

---

## ❌ DO NOT:

* Insert fake zero points (except start)
* Auto-normalize data

---

> “Not yet explored — something will happen here”

```text

```

---

## 📍 FUTURE START POINT:

* Begins immediately after:

  * Last real data point

### ❌ DO NOT:

* Continue line
* Show fake projection

---

* Line ends cleanly at last data point

### 2. SOFT GRADIENT FADE

### 3. BLUR OVERLAY (SUBTLE)

### 6. NO TOOLTIP ZONE

  * No tooltip OR
  * “No data yet” (very subtle)
