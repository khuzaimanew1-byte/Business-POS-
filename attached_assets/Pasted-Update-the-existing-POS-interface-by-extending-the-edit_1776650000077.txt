### 🧠 EDIT MODE (ENVIRONMENT REFINEMENT)

When entering edit mode via the pencil icon:

* The UI should clearly feel like an editing environment
* Product cards switch into editable state smoothly
* Keep transitions subtle and fast

For product cards in edit mode:

* Convert fields like name, price, and stock into input fields
* Keep the layout stable (no shifting or breaking)

Important:

* The quick code should remain visible as a label on the image
* The “#” (hashtag) must remain static and not part of the editable input
* Only the numeric/code value can be editable (if editing is allowed), but the “#” stays visually fixed

---

### 🗑️ PRODUCT DELETE FEATURE

Add a delete icon on each product card:

* Position: top-right corner of the card (overlay style, clean and minimal)
* Icon should be small but clearly clickable

Behavior:

* On click, trigger a confirmation popup before deleting
* If confirmed:

  * Remove product from UI
  * Also delete from database (or backend state)
* If canceled:

  * No changes occur

The delete action should feel safe and intentional, not accidental.

* Include a small “minus” icon inside a circle (–) within each category pill

Behavior:

* Clicking the delete icon triggers a confirmation popup

### ⚠️ DELETE CONFIRMATION SYSTEM (GLOBAL)

For both products and categories:

Whenever a delete action is triggered:

* Show a confirmation popup/modal

* Clear message like: “Are you sure you want to delete this?”

* Simple, centered modal

This ensures:

* No accidental deletions
* Better user confidence
