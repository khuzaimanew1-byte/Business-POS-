Design and implement a **dedicated Add Product Page** for the POS system. This page replaces the previous modal-based product creation and must feel like a **fast, professional product entry environment**, optimized for real retail workflows.

You are allowed to enhance design, spacing, animations, and interaction quality wherever needed. The system must feel clean, fast, and premium — not like a basic form.

---

# 🧭 GLOBAL STRUCTURE

* Use the **same global navigation bar** as the main POS page:

  * Desktop → left sidebar (icons only)
  * Mobile → bottom navigation bar

* Page should feel like part of the system, not a separate screen

---

# 🧱 CORE LAYOUT SYSTEM (DESKTOP / LAPTOP)

Create a **3-zone intelligent layout** optimized for speed and clarity:

---

## 1️⃣ LEFT — IMAGE ZONE (VISUAL ANCHOR)

* Square image upload container (fixed ratio)
* Centered circular plus icon
* Supports:

  * Click to upload
  * Drag & drop

### States:

* Empty → subtle placeholder
* Hover → soft border highlight
* Image selected → preview fills container
* Replace/remove via subtle overlay interaction

### Interaction:

* Dragging image over → soft magnetic highlight
* Image appears instantly after selection

This section should feel visually strong but not heavy.

---

## 2️⃣ CENTER — PRIMARY INPUT FLOW (MAIN FOCUS)

This is the **fast-entry core area**.

### Order (top → bottom):

### 🏷️ Product Name

* Required field
* Clean input with focus glow
* No duplicate warning (removed as per requirement)

---

### 🔖 Quick Code Input

* Starts with static `#` prefix (non-editable)
* User enters remaining value
* Must be **unique across products**
* If empty:

  * Auto-generate from product name (3–4 letters, readable format)

### Behavior:

* Live uniqueness validation
* If duplicate:

  * subtle warning
  * prevent submission

---

### 💰 Price + Profit (COMBINED ROW)

* Two inputs side-by-side
* Both required numeric fields
* Currency symbol fixed (non-editable)

---

### 📊 PROFIT LOGIC (STRICT + VISUAL)

* Profit **cannot exceed price**

### Behavior:

* If profit ≤ price:

  * normal state

* If profit > price:

  * input border turns red
  * subtle shake animation
  * inline message:
    “Profit cannot exceed price”

---

### 📈 LIVE MARGIN PREVIEW

Below price/profit:

* Show:

  * margin %
  * calculated value

* Updates in real time as user types

---

## 3️⃣ RIGHT — SECONDARY INPUTS

Lower priority, lighter visual weight:

### 📦 Stock Input

* Numeric only
* Clean minimal input

---

### 🗂️ Category Dropdown

* Dropdown with:

  * existing categories
  * last option: **“+ Add Category”**

### Behavior:

* Clicking “Add Category”:

  * inline mini input appears
  * create instantly
  * auto-select new category

---

# 🧩 VISUAL GROUPING

* Use soft spacing instead of heavy borders
* Light card-like grouping allowed (very subtle)
* Avoid boxed/cluttered UI

---

# 🔻 BOTTOM — ACTION ZONE

Two buttons:

### ✅ Create

* Creates product
* Shows quick success feedback
* Redirects to main POS page

---

### ➕ Create & Add Another

* Creates product
* Smoothly resets form
* Keeps user on same page

---

### ✨ SUBMIT INTERACTION

* Button → loading state
* Subtle success tick
* Then action (redirect/reset)

---

# ⚡ FLOW OPTIMIZATION

User flow should feel natural:

Image → Name → Quick Code → Price/Profit → Stock → Category → Create

* No confusion
* No unnecessary movement
* Optimized for keyboard + speed

---

# ⌨️ KEYBOARD SHORTCUT SYSTEM

Implement global shortcuts:

* **Shift + P**
  → Open Add Product page

* **Enter**
  → Create + Redirect

* **Shift + Enter**
  → Create & Add Another

* **Shift + A**
  → Open Analytics page

  → Toggle search bar focus/unfocus
