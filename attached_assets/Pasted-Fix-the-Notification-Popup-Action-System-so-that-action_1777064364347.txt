Fix the **Notification Popup + Action System** so that action buttons always render correctly and navigation takes the user precisely to the target product with a clear visual highlight.

This is a **bug fix + UX correction**, not a redesign.

---

# 🔔 🎯 ISSUE 1 — ACTION BUTTON NOT VISIBLE IN POPUP

## ❌ Problem:

* Notification toast appears
* But **action button is missing or not rendered**

---

## ✅ FIX: FORCE ACTION BUTTON RENDER

### Ensure:

* Every notification object includes:

  * `actionLabel`
  * `actionTarget` (quickCode or product id)

---

### UI Rule:

If notification is actionable:

```text
if (actionLabel && actionTarget)
  renderActionButton = true
```

---

### 🔘 Button Placement (Popup):

```
[ Message Text        ]  [ Action Button ]
```

* Right aligned
* Always visible (no overflow hidden)
* No conditional hiding unless truly non-actionable

---

### 🎨 Style Fix:

* Ensure:

  * `display: flex`
  * `justify-content: space-between`
  * Button not pushed out due to text overflow

---

### ⚠️ Common Bug Fixes:

* Remove:

  * `overflow: hidden` blocking button
  * fixed width causing clipping
* Add:

  * `min-width` for button
  * text wrapping instead of pushing button out

---

# 🔗 🎯 ISSUE 2 — ACTION CLICK NOT NAVIGATING TO EXACT PRODUCT

---

## ❌ Problem:

* Click works or not
* But does NOT go to exact product
* Or product visible nahi hota

---

## ✅ FIX: PRECISE NAVIGATION SYSTEM

---

### STEP 1 — IDENTIFY TARGET

Each notification must carry:

```json
{
  "quickCode": "#abc",
  "productId": "optional"
}
```

---

### STEP 2 — NAVIGATE TO PAGE

```text
if (not on main page)
  navigateToMainPage()
```

---

### STEP 3 — WAIT FOR RENDER

IMPORTANT:

```text
wait until product grid is mounted
```

Use:

* `setTimeout` OR
* `requestAnimationFrame` OR
* state-ready callback

---

### STEP 4 — FIND PRODUCT ELEMENT

```text
querySelector(`[data-quick-code="#abc"]`)
```

---

### STEP 5 — SCROLL INTO VIEW

```javascript
element.scrollIntoView({
  behavior: "smooth",
  block: "center"
})
```

---

### 🎯 RESULT:

* Product screen ke beech mein visible ho

---

# ✨ 🎬 ISSUE 3 — HIGHLIGHT ANIMATION MISSING

---

## ✅ ADD: PRODUCT HIGHLIGHT EFFECT

---

### Behavior:

After scroll:

```text
applyHighlightClass(element)
```

---

### 🎨 VISUAL EFFECT:

* Soft glow border
* Slight scale (1.02)
* Fade-out after 1.5–2s

---

### Example CSS:

```css
.highlight-product {
  animation: highlightPulse 1.6s ease-out;
}

@keyframes highlightPulse {
  0% {
    box-shadow: 0 0 0px rgba(255,255,255,0);
    transform: scale(1);
  }
  40% {
    box-shadow: 0 0 12px rgba(255,255,255,0.4);
    transform: scale(1.02);
  }
  100% {
    box-shadow: 0 0 0px rgba(255,255,255,0);
    transform: scale(1);
  }
}
```

---

### Remove class after animation:

```javascript
setTimeout(() => {
  element.classList.remove("highlight-product")
}, 1800)
```

* fallback scroll to category

---

## CASE 2: USER ALREADY ON PRODUCT

* Skip navigation
* Just highlight

---

## CASE 3: MULTIPLE MATCHES

* Pick first visible OR exact match
